React Basics

Short development guide for React in context of web development at ZIM Graz.

Prerequisites

Installation Requirements
  1. git (can be installed via GUI-Tool like tortoise git). And setup of git user-credentials (username + email to format commit messages)
  2. Visual Studio Code (Recommended extensions: "Material Icon Theme", "Prettier", "TSLint", "XML Tools", "Stardog RDF Languages Extension Pack", "Python")
  3. Node LTS version.
Setup

1. Clone or fork "ZIM React Template" from gitlab (make sure that you have an account and being assigned to the frontend gitlab-group --- Assignment needs to be done by a sysadmin)

2. Either work locally or create an own repository with the template code content.

3. Switch to root directory and run "npm ci" in the command line to install defined dependencies in package.json.

Verify template code

Check default behavior of cloned template code.

  1. In the root directory if your cloned template run "npm run start" to start the development server
  2. On http://localhost/3000 you should be able to access the template's html code

Cheatsheet

Common concepts
# Name Related Keywords Description
1 React Hooks React 16.8, no-class, functional components From React Docs: Hooks are a new addition in React 16.8. They let you use state and other essential React features without writing a class. Be careful to not to confuse React Hooks with the outdated React Lifecycle Hooks.
2 state rerender, set-state, dom-manipulation Instead of direct dom-manipulation like in JQuery, React tracks state changes and applies these changes to the DOM on it's own.
3 side effects ajax, request data Vague term describing mainly data-changes applied by asynchronous operations (usually followed by DOM changes). Like user-input, requests, etc.
4 Component Lifecycle componentDidMount, mount, unmount Before the React Hooks API the class based React components had a well defined lifecycle to which hook-functions coud be applied. Like componentDidMount, or beforeUnmount.
5 Component module pattern, encapsulation, abstraction Like the other big Shadow-Dom libraries React requires the developer to create components. This way React is able to enforce some important aspects of software-design. Like encaspulation and abstraction via the module-pattern, promoting the reuse, testability and maintainability of Javascript code.

ZIM Conventions

About collaboratively adopted dependencies and workflows

Description of dependencies where other implementations might exist but a ZIM wide agreement guarantees exchangability, reuse, maintainability and testability of code. Therefore we strongly recommend to follow the guidelines from doown below:

  1. Use the ZIM React template via gitlab (as opposed to many uncomparable React setups at the ZIM via individual Create React App setups)
  2. Use React Hooks (as opposed to class based components as before React 16.8)
  3. Use Typescript (as opposed to just writing Vanilla JS to promote exchange and reuse)
  4. Use Webpack (as opposed to Rollup.js or other build tools to guarantee similar repo structures)
  5. Use npm (e.g. Yarn)
  6. Avoid React Router (as opposed to interfering with the defined REST routes of GAMS)
  7. Avoid Redux (as opposed to building simple and maintainable web apps)

GAMS Usage

Development

Clone the ZIM React Template, copy over files and develop locally (via node) on your machine.

The ZIMReactTemplate comes with an own test-server via node.js

If the app builds intern links make sure to use environment variables to specify server names etc.

Production - Deployment to glossa

Same as for glossa --- make sure that servernames etc. are correct.

Deployment to GAMS

Basically you need to copy over the static files (inside "static"-folder) created in the "dist" folder to your gamsdev and link them with link and script tags.

Copy over all built (via "npm run build") .js and .css files from the static folder to your gamsdev and include it via script / link tags.

Don't forget to specify a container for your react app to render onto. Usually a div with id = "app"Change that default id via changing the id selector inside /src/index.tsx
