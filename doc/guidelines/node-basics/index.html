Node Basics

Basic Guideline for the use of Node.js as frontend development tool in context of Digital Editions.

Prerequisites

Installation Requirements
  1. There are no required dependencies for the use of node.js
  2. Recommended is to install VSCode to work with node.js
  3. In this Guideline node.js is described as frontend-development tool (compiling js / scss etc.) and not as Backend related technology. (Setting up http API)
Setup
  1. Install Node.js LTS version 12.x on your machine. (Googling node LTS should do the trick) With 12.x you should be able to run all ZIM related frontend code. Use higher versions at your own risk.

Quickstart

Demo
  1. Take a look at the React template in gitlab or the Docu Client itself.
  2. Clone one of these repos and run "npm i" --- npm will then install all required dependencies from npmjs (=registry)

Cheatsheet

Common concepts
# Name Related Keywords Description
1 Node.js Chrome v8 Engine, Node Node is a runtime environment for executing JS code. Essentially, Node is a C++ program that embeds Chrome’s v8 engine, the fastest JS engine in the world. With Node you can build fast and scalable networking applications. It’s a common choice for building RESTful services. BUT: It also established itself as somewhat standard development tool for frontend-technologies.
2 NPM Node Package Manager, Dependency Management Npm allows to install, update, version etc. dependencies via the package.json. It therefor provides transparency which dependencies are used in which version in given code repo. Npm is the world's largest Software Registry. npm is installed with Node.js.
3 npmjs.com NPM Registry, download dependencies. Web platform of npm that centralizes the exchange and reuse of code among Fronted Developers.
4 package.json dependency management File in the root of every npm-node repository. Clarifies which dependencies (in which version) are needed to make the application run on the current machine.
Frequently needed CLI Commands

npm i

Install all dependencies stated in the package.json

npm i

Install a single dependency from npmjs (registry)

npm i

Install a single dependecy at specified version-number

npm i --save-dev

Install a single dependecy as development dependency

npm un

Uninstall a single dependecy.

npm outdated

View outdated dependencies according to package.json

npm update

Update dependencies
